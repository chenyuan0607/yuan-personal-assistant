# CloudBase 临时中转站实施计划

**目标：** 在不改动线上 EdgeOne 部署的前提下，为 CloudBase 免费环境部署兼容的反馈和聊天 API，并通过配置随时切回 EdgeOne。

**方案：** 保持浏览器端 API 契约不变，新增 CloudBase HTTP 云函数和文档型数据库适配层。CloudBase 端复用现有校验、认证、反馈键和模型调用规则；部署参数与密钥只存在服务端环境变量。电脑端继续通过可配置的 API 地址收取反馈。

## 任务 1：建立跨厂商存储契约

- 为现有 KV 操作提取最小存储接口：`get`、`put`、`delete`、`list`。
- 新增内存适配器测试，覆盖幂等写入、前缀读取和删除。
- 保证 EdgeOne 现有存储绑定继续通过全部测试。

## 任务 2：实现 CloudBase 文档数据库适配器

- 新增 CloudBase 服务端目录和文档数据库适配器。
- 将稳定业务键映射为文档唯一键，支持按前缀和日期查询。
- 先写失败测试，再实现写入、覆盖、读取、确认和错误重试。

## 任务 3：部署最小反馈 API

- 在 CloudBase 云函数中接入现有认证、反馈白名单和 Codex 拉取/确认逻辑。
- 配置明确的测试来源跨域白名单。
- 验证敏感账本字段被拒绝，本地写入失败时云端保持待处理。

## 任务 4：接入聊天与 AI 模型

- 复用现有聊天记录格式和登录令牌格式。
- 新增 OpenAI 兼容模型适配配置：接口地址、模型名称、API 密钥。
- 限制发送给模型的近期消息数量和总长度。
- 测试真实 AI 回复、无效密钥错误和日志脱敏。

## 任务 5：加入可切换测试入口

- 不修改线上 EdgeOne 默认地址。
- 增加本地或独立测试入口，将助手 API 指向 CloudBase HTTP 地址。
- 验证切换地址不影响记账、任务、番茄钟和断网队列。

## 任务 6：部署与端到端验收

- 在 `yuan-assistant-test` 创建所需集合、云函数和环境变量占位。
- 由用户本人在 CloudBase 页面粘贴 AI API 密钥。
- 部署测试函数，产生一条任务反馈和一段真实 AI 对话。
- 运行电脑端收取，确认本地报告成功后云端才标记处理。
- 运行完整测试并记录 CloudBase 测试地址和回退方法。

## 停止条件

- 页面出现非 0 元费用、自动续费或按量计费时停止并征求确认。
- 需要验证码、密钥粘贴或新的账号权限授权时交给用户操作。
- 不自动删除 CloudBase 环境或测试数据。
