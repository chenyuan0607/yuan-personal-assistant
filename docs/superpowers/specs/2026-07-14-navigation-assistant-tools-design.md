# 导航与助手工具设计

日期：2026-07-14

## 目标

简化个人助理网页的顶部和助手界面，把底部导航重组为“今天、助手、账本、其他”，并建立一个可持续扩展的“其他”功能入口页。新增天气、语音录入、本地头像和表情包入口，不改变现有 CloudBase 对话、同步和隐私化反馈协议。

## 页面结构

删除全站顶部“个人小助手 / 缘的今天 / 账目仅存本机”栏。底部导航从左到右调整为“今天、助手、账本、其他”，网页首次打开仍显示“今天”。

原“回顾”页面改为“其他”入口页。入口页顶部直接展示天气，下面使用功能入口网格。第一批入口包括：

- 成长回顾：进入保留原有周总结和最近七天内容的独立子视图。
- AI 充值：在新标签页打开 `https://synthapi.asia/wallet`。
- 后续功能：继续以同一规格的入口添加，不把所有功能内容堆在“其他”首页。

## 天气

默认位置为贵州省六盘水市六枝特区。天气模块显示当前温度、天气状况、今日最高/最低温度和未来三天预报，并提供刷新、修改城市和主动定位按钮。

城市选择保存在当前浏览器。定位仅在用户点击定位按钮后请求浏览器位置权限，不在后台持续追踪。天气数据使用无需密钥的 HTTPS 天气接口；接口失败时显示“天气暂不可用”，不影响其他页面。

## AI 助手

删除“临时中转资料”界面。保留聊天登录、消息、同步和输入区。

助手标题区域增加头像。点击头像可以从当前设备选择图片；图片缩放为适合网页显示的尺寸后保存在当前浏览器，不上传 CloudBase，换设备后需要重新设置。

输入区增加麦克风图标按钮。支持语音识别的浏览器中，按住按钮开始识别，松开后把中文识别结果追加到输入框，不自动发送。不支持时提示用户使用输入法语音；首次使用时由浏览器请求麦克风权限。

输入区增加表情图标按钮和空素材库面板。当前没有图片时显示空状态，文字聊天不受影响。后续用户提供素材后，用户可以手动选择，AI 只能通过受控表情标识选择素材，不在线抓取或生成图片。

## 隐私与兼容

- 头像、默认城市和用户界面偏好只保存在当前浏览器。
- 不保存录音文件，不把语音音频上传中转站。
- 天气定位只在用户主动点击时发生。
- 外部充值链接使用新标签页和安全的链接属性。
- 手机自带浏览器不支持网页语音时，打字和输入法语音仍正常使用。
- CloudBase、SynthAPI 密钥、访问码和聊天存储格式保持不变。

## 验收

1. 顶部栏和临时中转资料不再显示。
2. 底部导航顺序为“今天、助手、账本、其他”，默认打开今天。
3. 其他首页顶部显示六枝特区天气，下方显示成长回顾和 AI 充值入口。
4. 成长回顾可以进入和返回，AI 充值链接指向 SynthAPI 钱包的新标签页。
5. 头像可以选择、更换并在刷新后保留于当前设备。
6. 支持的浏览器可以按住语音转文字，不支持时有明确降级提示。
7. 表情入口在空素材库下可正常打开和关闭，不影响发送文字。
8. 桌面和 390px 手机宽度下无横向溢出、控件重叠或不可点击区域。
9. 全部自动测试通过，部署产物不包含任何秘密。

## 发布目标

本次改动完成验收后直接发布到现有正式 EdgeOne 网站。CloudBase 继续作为当前 AI 对话后台，不迁移数据或更换模型接口。发布前保留可回退的 Git 提交，并先验证构建产物和 CloudBase 跨域来源；不再把 CloudBase `assistant-test` 网址作为本次最终交付目标。
