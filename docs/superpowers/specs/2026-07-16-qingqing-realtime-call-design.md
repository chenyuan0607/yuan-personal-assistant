# 青青实时通话设计

## 目标

为个人助手网页的聊天页增加独立全屏通话模式。用户可持续用语音或文字与青青交流，青青以语音回应；通话产生的双方文字保留到现有聊天记录和夜间整理，不保存原始音频。

## 入口和页面

- 聊天页右上角新增电话按钮。
- 点击后进入独立全屏通话页；红色挂断按钮是唯一的正常退出方式。
- 顶部中央显示通话时长，左上角留空，右上角为眼睛按钮。
- 默认头像通话模式：中央显示青青头像、名字与当前一句话；头像外圈以轻微呼吸和音量起伏表示状态。
- 眼睛按钮切换字幕模式。字幕模式隐藏头像和名字，中央区域显示持续上滚的完整通话文字。

## 通话状态和控制

- 状态为正在听、正在想、正在说、正在重新连接。
- 底部由麦克风、挂断、钢笔三个圆形按钮组成。
- 麦克风开启时持续监听。用户停顿后进入青青回复；青青说完自动恢复监听；用户重新说话可打断青青。
- 钢笔按钮打开文字输入时，立即暂停麦克风监听；关闭文字输入后恢复持续监听。
- 文字输入发送后，青青仍以语音回复。

## 字幕和记录

- 字幕模式完整显示用户与青青双方产生的文字，按时间顺序向上滚动，并以简洁说话人标记区分。
- 默认头像模式仅显示当前正在说的那一句，避免遮挡头像主体。
- 每段可确认的用户语音转写、用户文字输入与青青回复均写入现有聊天记录。
- 夜间整理读取这些文字；不保存、上传或归档原始音频。

## 网络和权限

- 首次进入时请求麦克风权限；无权限时明确显示无法开始通话，不录制音频。
- 发生短暂网络或识别中断时，不自动挂断，暂停音频处理并显示正在重新连接，然后自动重试。
- 用户主动挂断，或连续重连超时后才结束通话。
- 页面前端不得持有实时语音服务的长期密钥。

## 服务结构

- 语音输入、语音输出和低延迟打断能力由国产实时语音 API 提供，优先评估与现有智谱账户兼容的实时能力。
- 现有青青文本模型继续提供人格、记忆包、任务和文字聊天逻辑。
- 新增受认证保护的实时中转服务，向浏览器提供短时会话能力并保护长期 API 密钥。
- 现有 CloudBase HTTP 中转继续承担文字记录与常规接口；实时中转承担持续音频会话。

## 验收

- 用户能从聊天页进入、挂断并返回原聊天页。
- 头像模式与字幕模式可随时切换，字幕包含双方文字。
- 语音与键盘输入都能得到青青的语音回复，文字进入聊天记录。
- 断网后进入重连状态，不立即结束通话。
- 原始音频不写入聊天记录、文件存储或夜间知识库。
