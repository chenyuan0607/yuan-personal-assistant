# 每日任务番茄钟与完成反馈设计

## 目标

在现有“今天”页面中，为每条每日任务增加按建议用时启动的番茄钟。用户在计时结束后确认“已完成”或“未完成”，网页保存专注记录并同步到 EdgeOne 中转站。网页同时在手机本地生成不含单笔明细的每日账本摘要。电脑端每天 00:10 一次性收取前一日的任务反馈、专注记录和账本摘要，整理后写入本地知识库，供 Codex 制定下一日计划时参考。

账本明细继续只保存在各自手机本地。中转站和 Codex 只能收到本地计算后的汇总数字，不能收到单笔账目、备注或账本备份。

## 用户流程

1. Codex 根据本地知识库生成每日任务，每项任务包含标题和建议分钟数。
2. “今天”页面在每条任务右侧显示建议时间和番茄钟按钮。
3. 用户点击按钮后进入独立计时视图，计时时长直接采用该任务的建议分钟数。
4. 用户可以开始、暂停、继续或提前结束计时。
5. 倒计时结束时，网页播放提示音并在设备允许时震动，然后要求用户确认“已完成”或“未完成”。
6. 确认结果先写入手机本地，再安静地同步到中转站；确认单项任务时不调用 AI，也不逐项唤醒 Codex。
7. 每次保存、修改或删除账目后，手机在本地重新计算当日隐私化摘要，并用同一个“设备＋日期”编号覆盖中转站中的旧摘要；不保留摘要版本历史。
8. 每天 00:10，电脑端一次性收取前一日的任务、专注记录和账本摘要，生成每日完成报告并写入本地知识库。
9. Codex 在制定下一日计划前读取报告，根据完成率、实际专注时间、消费摘要和多次未完成情况调整任务数量、大小、难度与建议时间。

## 今天页面

“今日焦点”下方增加“今日进度”区域：

- 进度条按已完成任务数除以当日任务总数计算。
- 显示百分比，例如 `75%`。
- 显示任务数量，例如 `已完成 3 / 4 项`。
- 同时显示当日所有有效专注记录累计的分钟数，例如 `今日专注 85 分钟`。
- “未完成”任务不增加完成百分比，但其有效专注时间计入累计专注时间。

每条任务包含：

- 任务标题。
- 建议分钟数。
- 番茄钟按钮。
- 当前状态：未开始、进行中、未完成或已完成。

已完成任务使用绿色状态；未完成任务使用橙色状态并允许再次启动番茄钟。

## 计时视图

计时视图显示：

- 当前任务名称。
- 任务建议专注时间。
- 大号剩余时间。
- 开始、暂停、继续和提前结束操作。

计时规则：

- 建议多少分钟就连续倒计时多少分钟，不自动拆成多个 25 分钟周期。
- 暂停期间不计入实际专注时间。
- 页面切换、刷新或短暂关闭后，根据目标结束时间恢复准确的剩余时间，避免浏览器后台节流造成倒计时误差。
- 浏览器或手机系统限制后台网页时，锁屏后的提示音与震动不保证触发；重新打开网页后仍能恢复正确状态并要求用户确认。
- 提前结束也进入结果确认，由用户决定本次任务是否已经完成。

## 完成确认与多次专注

每次计时结束后必须由用户选择：

- `已完成`：任务最终状态设为已完成，并记录完成时刻与本次实际专注时间。
- `未完成`：任务保持未完成，并记录本次实际专注时间；用户可以再次启动该任务。

同一任务可以有多次专注记录。后续选择“已完成”时，保留此前全部专注次数和有效专注时间。任务最终状态以最后一次明确确认结果为准。

## 本地存储与同步

手机本地保存：

- 当日任务快照。
- 每次专注的开始、暂停、恢复、结束及有效时长。
- 用户确认的已完成或未完成结果。
- 上传状态和设备名称。
- 完整账本明细、备注、预算设置和备份数据。

同步规则：

- 本地写入成功后再尝试上传，断网不阻塞用户确认。
- 断网记录进入待上传队列，恢复网络或下次打开网页时自动补传。
- 每条记录使用稳定的唯一编号，重复上传不会产生重复数据。
- 两台手机都可以记录，并在数据中标明来源设备。
- KV 审核未通过时，功能先以纯本地方式运行；KV 绑定完成后启用云端同步。

## 账本摘要与隐私边界

每日账本摘要由手机在本地从完整账本计算，只允许包含：

- 摘要日期和设备名称。
- 当日总收入、总支出和收支差额。
- 当日收入笔数和支出笔数。
- 当日各支出分类的合计金额。
- 当前月份的月总预算、截至当日的月累计支出、剩余预算和预算状态。

摘要不得包含：

- 单笔账目的金额、编号或创建时间。
- 任何账目备注。
- 单笔账目列表或可还原单笔账目的排列信息。
- 常用账目模板、完整自定义分类设置或账本备份文件。
- 手机本地 IndexedDB 中的原始账本记录。

两台手机的账本仍彼此独立，不进行账目同步。每台设备分别生成带设备名称的摘要；每日报告分设备展示摘要，避免把两个独立账本误认为同一个账本。摘要上传失败时保留在本机待上传队列，不影响记账。

手机网页不能保证在锁屏或关闭后于午夜自动运行，因此不依赖手机执行 00:10 定时任务。账本发生变化时立即在本地重算并覆盖当日摘要，打开网页时也会重试待上传摘要。电脑端 00:10 收取当时已经到达中转站的最新摘要；设备长期离线造成的迟到摘要在后续收取时补入对应日期的报告。

## 中转站与每日收取

中转站保存未经 AI 分析的结构化任务结果、专注记录和隐私化账本摘要。单项完成或摘要上传时只写数据，不触发模型调用。

电脑端每天 00:10 执行一次收取：

1. 拉取前一日尚未处理的任务、专注记录和账本摘要。
2. 按任务编号合并两台手机的记录并去重。
3. 计算任务总数、已完成数、未完成数、完成率、总专注时间和每项任务的专注次数。
4. 按设备整理当日收入、支出、分类合计和预算状态，不读取单笔账目。
5. 生成 Markdown 每日完成报告并写入 `D:\缘的成长知识库` 的待整理区域。
6. 确认本地文件安全写入后，将云端记录标记为已收取。
7. 已收取原始记录按既定保留期清理；重要的整理报告长期保留。

同一设备同一天的账本摘要在中转站只保留最新值。电脑端已收取后，如果该日期出现迟到或更新后的摘要，则重新生成该日期报告并保留更新时间说明，不重复创建多份互相冲突的报告。

## 每日完成报告

报告至少包含：

- 日期。
- 当日安排任务数、已完成数、未完成数和完成率。
- 当日总专注分钟数。
- 每项任务的标题、建议分钟数、实际专注分钟数、启动次数和最终状态。
- 每台设备的当日总收入、总支出、收支差额、分类支出合计和月预算状态。
- 可供下一日计划参考的客观信号，例如连续未完成、明显超时或多次启动后完成。

第一版只生成客观报告，不在 00:10 额外调用 AI。Codex 在下一次制定计划时读取报告并进行分析。

## 错误与安全处理

- 番茄钟、任务反馈与账本使用彼此独立的存储空间和代码路径；同步代码只能接收本地生成的账本摘要，不能读取或发送原始账本记录。
- 未成功上传的记录始终保留在手机本地。
- 未成功写入本地知识库的云端记录不能标记为已收取。
- 上传和收取接口均要求有效身份令牌。
- 网页不显示或记录 API 密钥、访问码与服务端签名密钥。
- 中转站接口拒绝包含账目备注、单笔账目数组或账本备份内容的摘要请求。
- 同步失败时使用清楚但不打断操作的状态提示，不把任务误标为完成。

## 第一版范围

包含：

- 任务旁的番茄钟入口。
- 独立计时视图。
- 开始、暂停、继续、提前结束。
- 结束后的已完成/未完成确认。
- 今日任务进度条和累计专注时间。
- 本地持久化、断网补传和中转站记录。
- 本地生成并上传不含单笔明细的每日账本摘要。
- 每天 00:10 的批量收取与本地知识库报告。

不包含：

- 排行榜、积分、徽章或连续打卡。
- 强制拆分成传统 25 分钟周期。
- 每完成一项就调用 AI。
- 锁屏状态下等同原生闹钟的保证。
- 两台手机之间实时同步正在运行的倒计时。
- 单笔账目、账目备注、账本备份或两个手机之间的账本同步。

## 测试重点

- 任务建议分钟数正确传入计时器。
- 暂停时间不计入实际专注时间。
- 刷新或重新打开页面后倒计时准确恢复。
- 已完成与未完成正确更新状态、进度条和累计专注时间。
- 同一任务多次专注正确累计且不重复。
- 断网记录保留并在恢复联网后补传。
- 两台设备上传的重复请求正确去重。
- 电脑端只有在本地报告写入成功后才确认收取。
- 账本摘要只包含允许的聚合字段，不能包含备注、单笔账目或备份内容。
- 两台设备的账本摘要保持分开并正确标注来源设备。
- 番茄钟功能不会读取、上传或更改账本明细。
