# CloudBase 独立测试网页设计

日期：2026-07-14

## 目标

部署一个独立测试网址，内容为当前个人助理完整网页。测试站保留任务、专注、账本、复盘和 AI 助手界面，但将助手 API 明确指向已经验证的 CloudBase 中转站。现有正式网址、现有 EdgeOne 后台和正式站数据保持不变。

## 方案

使用同一份前端代码生成测试站静态文件，在测试站入口的 `html` 元素上写入 CloudBase API 地址。网页继续通过现有 `createAssistantApi` 和 `initAssistant` 工作，不在业务代码里写死云厂商，也不增加用户可误触的后台切换按钮。

测试站部署到 CloudBase 静态托管并取得独立网址。CloudBase 云函数的跨域白名单只增加该测试网址，不开放任意来源。模型密钥、会话密钥和访问码哈希继续只保存在云函数环境变量中，不进入网页、Git 或部署产物。

## 用户体验

用户打开测试网址后看到与正式站一致的完整网页。进入“助手”页，首次使用时输入原有专属访问码和设备名称；登录成功后可以发送消息、接收 AI 回复，并在另一浏览器会话中读取相同日期的历史消息。

网络失败时沿用现有本地待发送队列，页面提示内容暂存在本机。恢复网络后再次进入助手页时自动重试。测试站使用独立浏览器存储空间，不改变正式网址中的本地任务和账本数据。

## 数据与隐私

- AI 对话上传当前问题、近期聊天和精简记忆，不上传整个知识库。
- 任务反馈和隐私化账本汇总沿用现有字段白名单。
- 单笔账目、备注、具体时间、账本模板和备份不上传。
- 测试站不展示或下发任何服务端密钥。
- 不自动删除 CloudBase 环境或测试数据。

## 部署与回退

先构建独立测试产物，再部署到现有 `yuan-assistant-test-d2bd198841e7` 免费 CloudBase 环境。部署前后均不修改正式 EdgeOne 站点。若测试失败，只停止使用测试网址或重新部署测试站，不需要回滚正式站。

未来 EdgeOne KV 审核通过后，先确认 CloudBase 中所有待处理记录已收取，再通过部署配置把正式站连接回目标中转站；本次测试站不自动执行该切换。

## 验收标准

1. 独立测试网址可在桌面和手机宽度下正常打开完整网页。
2. 正式网址和正式部署保持不变。
3. 使用原访问码可以登录测试站助手。
4. 发送测试问题后，页面显示 `deepseek-v4-flash` 的真实回复。
5. 刷新页面后仍可读取同一天的对话记录。
6. 第二个独立浏览器会话可以读取并继续同一段对话。
7. 浏览器控制台没有导致核心流程中断的错误，页面控件没有重叠。
8. 全部自动测试通过，部署产物中不含密钥、访问码或令牌。
