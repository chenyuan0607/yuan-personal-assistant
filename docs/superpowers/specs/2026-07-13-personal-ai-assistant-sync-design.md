# 个人 AI 助手与知识库同步设计

日期：2026-07-13

## 1. 目标

在现有个人网页中增加一个只供本人使用的 AI 助手，使两台手机可以连续聊天，并建立以下资料循环：

1. 手机产生聊天、文字、图片和普通文档等原始资料。
2. EdgeOne Makers 作为可删除的临时中转站。
3. Codex 从中转站接收资料，保存到电脑本地待整理区。
4. Codex 整理资料并写入 `D:\缘的成长知识库`。
5. Codex 生成精简记忆包，供网页 AI 次日读取。
6. Codex 将整理完成、适合长期保存的正式资料上传到 IMA。
7. IMA 只作为精选云端知识库和备份，不保存琐碎聊天原文。

本地知识库是完整、长期的主仓库；EdgeOne Makers 是临时中转站；IMA 是整理后资料的云端备份。

## 2. 已确认的边界

- 账本继续只保存在每台手机的浏览器本地，不同步、不上传、不发送给 AI。
- 两台手机使用同一个本人账号，可以查看并继续同一天的聊天。
- 电脑不需要同步聊天界面，只需要 Codex 接收并整理资料。
- 网页 AI 平时只聊天，不自动把零碎内容写入长期知识库。
- 睡前可以选择“查看今日整理”或“直接归档并休息”。
- 原始聊天和临时文件不经过 IMA。
- IMA 中的资料默认不自动删除。
- WorkBuddy 最终退出，由 Codex 接管接收、整理和上传工作。

## 3. 总体架构

```text
两台手机上的个人网页
        ↓
EdgeOne Makers 私人中转站
  ├─ Functions：身份验证、聊天、上传下载、删除
  ├─ KV：聊天文字、状态、记忆包版本
  └─ Blob：图片、文档和其他临时文件
        ↓
电脑上的 Codex
  ├─ 下载原始资料
  ├─ 保存本地待整理区
  ├─ 分析、分类和归档
  ├─ 生成精简记忆包并上传 EdgeOne
  └─ 上传正式资料到 IMA
        ↓
D:\缘的成长知识库（长期主仓库）
```

EdgeOne Makers 是当前主服务。中转接口保持服务商无关，未来可迁移到 Cloudflare 或 CloudBase，而不重做网页和 Codex 工作流。

## 4. 网页功能

现有网页增加“AI 助手”页面，提供：

- 当天聊天窗口。
- 两台手机的聊天同步。
- 文字消息。
- 上传文字、图片、PDF、Word、Markdown 和普通文本文件。
- 单文件第一版限制为 20 MB，不支持视频。
- “查看今日整理”按钮：生成预览，确认后归档。
- “直接归档并休息”按钮：生成后直接进入归档流程。
- 中转资料状态：等待接收、Codex 已接收、已整理、等待删除、已删除、处理失败。
- “标记长期保留”“立即删除”“重新处理”操作。
- 当前记忆包版本和最近更新时间。

账本页面与 AI 助手页面在数据和代码路径上隔离。AI 助手不得读取账本的 IndexedDB 数据。

## 5. 身份与设备

- 系统第一版只有一个本人账号，不开放注册。
- 两台手机首次使用时输入同一个专属访问码。
- 验证成功后签发设备会话，默认 90 天重新验证一次。
- 支持主动退出和让丢失设备失效。
- 访问码不写入网页源代码、GitHub、日志或知识库。
- Codex 使用独立的电脑端 API Token，不与手机访问码共用。

## 6. 聊天与长期记忆

记忆分为三层：

1. 当天上下文：当天完整聊天，临时保存在 EdgeOne。
2. 近期记忆：最近几天的精简总结。
3. 长期记忆：Codex 根据本地知识库生成的“手机助手记忆包”。

Codex 每次生成带版本号和生成时间的记忆包。EdgeOne 保留最近 7 个版本。网页 AI 每天首次打开时读取最新版，同一天缓存使用，不在每条消息前重复读取。

记忆包只包含手机助手确实需要的内容：沟通偏好、当前目标、近期状态、重要规则、待跟进事项。完整本地原始资料不得作为记忆包上传。

## 7. 资料生命周期

### 7.1 原始聊天

1. 两台手机的原始聊天进入 EdgeOne KV。
2. 用户触发“查看今日整理”或“直接归档并休息”。
3. 网页 AI 生成今日整理结果。
4. Codex 在次日 08:30 前后接收原始资料和整理结果。
5. Codex 生成或完善正式本地文档。
6. 正式本地文档可上传 IMA。
7. 全部必要步骤成功后，原始聊天进入 7 天删除倒计时。

### 7.2 临时文件

1. 文件上传 EdgeOne Blob，状态为“等待接收”。
2. Codex 下载后校验文件大小和内容哈希。
3. 校验成功后保存到本地待整理区。
4. Codex 处理完成并记录正式输出位置。
5. 临时文件进入 7 天待删除期。
6. 7 天内可恢复或标记长期保留；到期后从 EdgeOne 删除。

尚未被 Codex 成功接收的资料不进入 7 天删除倒计时，最多保留 30 天并在网页提示。

### 7.3 正式资料

随笔、感悟、计划、目标、正式文档和 Codex 的最终整理结果长期保存在本地知识库。经规则允许后，Codex 将其上传 IMA。IMA 中的正式资料不由系统自动删除。

## 8. 本地目录建议

```text
D:\缘的成长知识库\
├─ 00-inbox\edgeone待整理\
├─ 05-projects\daily-plans\
├─ 05-projects\daily-conversations\
├─ 05-projects\mobile-assistant\
│  └─ 手机助手记忆包.md
└─ .system\assistant-sync\
   ├─ processing-temp\
   ├─ pending-delete\
   ├─ sync-state.json
   └─ audit-log.jsonl
```

`sync-state.json` 保存资料唯一编号、哈希、接收时间、处理状态、本地目标位置和 IMA 上传状态，防止重复下载和重复上传。日志只记录元数据，不复制敏感正文。

## 9. Codex 工作流

Codex 接管 WorkBuddy 的职责：

- 每天 08:30 检查 EdgeOne 新资料。
- 下载并校验未接收的聊天和文件。
- 将原始资料保存到本地待整理区。
- 整理、分类并补充本地知识库。
- 生成最新版手机助手记忆包并上传 EdgeOne。
- 将允许上传的正式文档写入 IMA。
- 记录同步状态并处理重试。
- 为 09:10 的每日任务生成流程提供最新本地资料。

第一阶段保留 WorkBuddy 并行运行 7 天，但只做验证，不允许两者同时写入相同本地目标。确认 Codex 无漏项、重复和乱码后停用 WorkBuddy。

## 10. IMA 集成

Codex 使用 IMA 官方 Client ID 和 API Key，通过官方 OpenAPI：

- 搜索和读取 IMA 笔记、知识库。
- 新建或追加正式笔记。
- 上传整理完成的正式资料。

IMA 密钥保存在电脑本地私密配置区，不进入项目仓库。Codex 默认只操作指定知识库或文件夹，不自动删除 IMA 内容。

由于 IMA 公开接口没有可靠的自动删除能力，所有需要到期删除的内容必须留在 EdgeOne，不得先上传 IMA。

## 11. AI 模型

第一版优先测试 EdgeOne Makers 当前提供的免费模型额度。聊天模型通过统一接口调用，可随时切换到 DeepSeek、通义、Kimi 或其他兼容模型。

模型只接收完成当前回答所需的聊天上下文、精简记忆和相关资料，不接收整个本地知识库。切换模型不得改变聊天记录格式、记忆包格式和中转接口。

## 12. 隐私与安全

- 所有网络传输使用 HTTPS。
- AI、IMA 和 EdgeOne 的密钥只存放在服务端或电脑本地秘密存储。
- 手机端只保存短期会话令牌，不保存服务端密钥。
- 文件使用不可猜测的随机编号。
- 上传和下载接口执行身份验证、大小限制、类型检查和频率限制。
- 账本永不进入 AI、中转站或 IMA。
- 聊天内容会经过所选 AI 模型供应商，这是实现云端 AI 对话的必要数据流。
- 第一版不实现聊天内容的端到端加密，因为 AI 服务需要读取正文；普通临时文件后续可增加手机端加密、电脑端解密。
- 支持立即撤销手机会话和 Codex Token。

## 13. 失败保护

- 手机消息先保存在本机待发送队列，服务端确认后才标记已同步。
- 网络失败自动重试，不覆盖旧聊天。
- Codex 未成功下载或校验失败时，EdgeOne 不删除原件。
- IMA 上传失败时，本地正式文档保留并进入重试队列。
- 同一资料通过唯一编号和哈希去重。
- 记忆包保留最近 7 个版本，可回退。
- 删除任务只处理已经完成全部检查且超过保留期的资料。
- 主服务故障时停止删除任务，并在网页显示状态。

## 14. 第一版范围

第一版必须完成：

- EdgeOne Makers 项目与私密 API。
- 单人访问码和两台手机设备会话。
- 文字聊天和跨手机同步。
- EdgeOne 免费模型或可替换 AI 接口。
- 每日整理预览与直接归档。
- 文字、图片和普通文档上传。
- KV/Blob 临时保存和删除规则。
- Codex 下载、去重、整理、记忆包上传。
- Codex 上传正式资料到 IMA。
- 状态展示、失败重试和审计记录。

第一版不包括：

- 视频上传。
- 多用户和朋友账号。
- 账本云同步或 AI 读取账本。
- 自动修改或删除 IMA 资料。
- 语音实时通话。
- 手机推送通知。

## 15. 后续可扩展功能

- 手机录音转文字和每日语音日记。
- 图片 OCR、票据和学习资料整理。
- 重要消息一键标记。
- 基于本地知识库的主题检索。
- 情绪、习惯、目标变化趋势。
- 每周和每月成长报告。
- 截止日期提醒和手机通知。
- 可选择的账本月度摘要，但必须由用户明确开启，并且默认只发送汇总数据。
- 中转服务一键迁移到 Cloudflare 或 CloudBase。

## 16. 成本与服务风险

EdgeOne Makers 当前提供免费版，额度包括 Functions、KV、Blob 和有限模型调用，足够个人第一版使用。官方说明商业化后配额和价格可能调整，因此系统必须保持可迁移。

第一版预期新增固定费用为 0 元。若免费模型质量不足，外部 AI 模型预计产生少量按量费用。IMA 继续使用用户现有账号和会员能力。

## 17. 验收标准

- 手机 A 发出的消息能在手机 B 刷新或重新打开后继续显示和对话。
- 两台手机只共享 AI 聊天和中转资料，不能读取对方手机本地保存的账本、预算和任务历史。
- 未验证设备不能读取聊天、文件和记忆包。
- Codex 能下载 EdgeOne 新资料且不会重复处理。
- Codex 能生成本地正式文档和最新版记忆包。
- 网页 AI 次日能读取新记忆包并体现其中的偏好和目标。
- Codex 能将指定正式资料上传 IMA。
- 原始聊天和临时文件只有在处理成功后才进入删除倒计时。
- 删除失败可重试，且不会误删未处理资料。
- WorkBuddy 停用后，完整资料循环仍能连续运行 7 天。
