# CloudBase 临时中转站设计

日期：2026-07-14

## 目标

在 EdgeOne KV 审核期间，使用腾讯 CloudBase 免费体验环境完成个人助理的最小中转闭环。EdgeOne 现有实现、线上网页和本地数据保持不变；EdgeOne KV 可用后，通过配置切换回原中转站。

## 范围

第一阶段只验证以下能力：

1. 手机提交任务结果和隐私化账本汇总。
2. CloudBase 安全保存待处理反馈。
3. 家中电脑读取反馈并写入本地知识库。
4. 本地报告写入成功后，云端记录才标记为已处理。

聊天接续、文件上传和模型调用不纳入第一阶段。最小闭环稳定后再决定是否临时迁移这些能力。

## 隐私边界

- 单笔账目、备注、具体时间、账本模板和账本备份不得上传。
- 只允许上传既有白名单中的任务反馈和隐私化账本汇总。
- CloudBase 环境不保存本地知识库全文。
- 密钥和访问令牌只保存在本地秘密配置或 CloudBase 服务端环境变量中，不写入 Git、聊天或知识库。
- CloudBase 测试数据在成功收取并确认后才允许清理。

## 架构

网页继续通过现有 `createAssistantApi` 调用统一 HTTP 接口。接口地址由部署配置选择：

- `edgeone`：现有 EdgeOne Functions + EdgeOne KV。
- `cloudbase`：CloudBase 云函数 + Serverless 文档型数据库。

CloudBase 云函数保持与现有反馈接口相同的请求和响应格式。电脑端同步脚本改用中立的中转站名称和可配置 API 地址，不在业务逻辑中判断具体厂商。

CloudBase 数据库使用一条文档表示一条反馈，稳定业务键作为文档唯一标识，保留 `waiting` 和 `processed` 状态。重复上传同一业务键时更新同一条记录，保证幂等。

## 切换与回退

线上 EdgeOne 页面在测试期间不改动。CloudBase 使用独立测试入口或显式 API 地址进行验证。

切回 EdgeOne 前：

1. 停止向 CloudBase 写入新数据。
2. 收取并确认所有 `waiting` 记录。
3. 将网页 API 地址切回 EdgeOne。
4. 验证 EdgeOne 写入、读取和确认闭环。
5. 保留 CloudBase 环境一段观察期，再决定是否停用；不自动删除数据或环境。

## 安全与错误处理

- 手机写入继续使用短期设备令牌；电脑读取和确认使用独立 Codex 令牌。
- 服务端继续执行字段白名单、长度、日期和金额范围校验。
- 浏览器断网队列保持现有行为，只有服务器确认成功后才删除本地待上传项。
- 云端读取失败、本地写入失败或确认失败时，记录保持 `waiting`，下次继续处理。
- CloudBase 跨域只允许明确配置的测试来源，不使用任意来源通配符承载带凭证请求。

## 测试与验收

1. 现有 58 项测试继续通过。
2. 新增 CloudBase 存储适配器单元测试：写入、覆盖、按日期读取、确认和失败重试。
3. 验证敏感账本字段被拒绝。
4. 在测试入口产生一条任务结果和一份账本汇总。
5. 运行电脑端同步，确认本地报告生成后云端才变为 `processed`。
6. 验证将 API 地址切回 EdgeOne 不需要改动网页业务代码。
